# Universal Portfolio Shrinkage Approximation

> Example generated by Copilot GPT-5.4.

Paper: Kelly, Malamud, Pourmohammadi, and Trojani (2025), NBER working paper.

## Methodology

Universal Portfolio Shrinkage Approximation (UPSA) constructs a nonlinear spectral shrinkage portfolio as a non-negative ensemble of ridge portfolios. Each month, the method estimates an uncentered second-moment matrix from a rolling training window, computes ridge portfolios over a grid of ridge penalties, estimates ridge-portfolio moments with leave-one-out cross-validation, solves a non-negative Markowitz problem in ridge space, and back-projects the ridge ensemble into factor-portfolio weights.

## Data Description

The documented paper2code example uses monthly JKP factor portfolio returns in wide CSV form, with one `date` column and one return column per factor. A companion reference file can contain UPSA weights by fit date for diff-based validation.

## Signal Logic

UPSA is not a price-trigger strategy. It is a monthly portfolio-construction strategy over factor returns. Weights are estimated at the end of month `t` and applied to factor returns in month `t+1`.

## Key Implementation Details

- Rolling lookback: 120 monthly observations.
- Second moment: uncentered matrix `F'F / T`, not mean-centered covariance.
- Ridge grid: powers of ten from `1e-10` through `1e-1`.
- LOO: within each rolling window, with finite-sample correction for `T-1` refits.
- Ensemble: non-negative weights over ridge portfolios.
- Final factor weights: long-short allowed, no per-factor clipping.
- Evaluation: out-of-sample starts after the first full training window.

## Paper Trading Note

This example consumes factor return series and produces factor-portfolio weights. It is a research/backtest contract, not a directly broker-connected paper trading strategy without an additional tradable-instrument mapping layer.
