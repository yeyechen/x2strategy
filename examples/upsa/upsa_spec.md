# Universal Portfolio Shrinkage Approximation (UPSA)

> Example generated by Copilot GPT-5.4.

**Type**: portfolio_optimization  
**Asset Class**: factor_portfolios

A monthly factor-portfolio construction strategy that approximates nonlinear spectral shrinkage by combining ridge portfolios with non-negative ensemble weights.

## Data Requirements

- **Input**: monthly factor return panel with columns `[date, factor_1, ..., factor_N]`
- **Frequency**: monthly
- **Lookback**: 120 months
- **Output**: monthly factor weights and strategy returns
- **Data semantics**: return series, not OHLC price bars

## Expected Metrics

Spec2Code should report:

- Sharpe ratio
- Maximum drawdown
- Total return
- Return value / final portfolio value

## Indicators / Inputs

| ID | Name | Category | Formula | Scope |
|:---|:-----|:---------|:--------|:------|
| `second_moment` | Uncentered second moment | matrix | `Sigma_bar = F'F / T` | rolling_window |
| `sample_mean` | Rolling sample mean | vector | `mu_bar = mean(F)` | rolling_window |
| `ridge_grid` | Ridge penalty grid | parameter | `{10^-10, ..., 10^-1}` | global |
| `loo_ridge_returns` | LOO ridge returns | matrix | T leave-one-out ridge return series | rolling_window |

## Logic Pipeline

1. Compute rolling uncentered second moment and sample mean.
2. Compute eigendecomposition for spectral arithmetic.
3. Compute ridge portfolios for all penalties in the grid.
4. Estimate leave-one-out ridge return moments inside the rolling window.
5. Solve the non-negative ensemble Markowitz problem in ridge space.
6. Back-project the ensemble into factor weights.
7. Apply trace-preserving variance normalization when required by the selected spec.
8. Apply weights with a one-month lag to construct realized strategy returns.

## Execution

Monthly rebalance. Weights estimated at month-end `t` are applied to month `t+1` factor returns.

## Paper Trading Note

This example is not directly broker-connected because its input is factor returns and its output is factor-portfolio weights. A broker-connected version requires a separate mapping from factors to tradable instruments or a different live-trading strategy contract.
